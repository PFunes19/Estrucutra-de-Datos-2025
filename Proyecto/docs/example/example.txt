1. Descripción del problema
En un call center de soporte técnico, las llamadas de los clientes deben ser atendidas en el mismo orden en que llegan.
Cuando un cliente llama:
Si hay agentes disponibles, se le asigna uno.
Si todos los agentes están ocupados, la llamada “espera en la fila”.
Cuando un agente termina una llamada, toma a la siguiente persona que estaba esperando.
Este comportamiento corresponde a “primero en entrar, primero en salir” (FIFO), lo que sugiere el uso de una cola (queue) como estructura de datos principal.
El objetivo del ejemplo es mostrar cómo el asistente de estructuras de datos identifica que la mejor estructura para este problema es una cola y cómo se aplican las operaciones básicas sobre ella.

2. Entradas del sistema
2.1 Entradas del cuestionario
El usuario (desarrollador del sistema) responde las preguntas del asistente pensando en el escenario del call center:
¿Necesitas acceder rápidamente a los elementos por su posición (índice)?
Respuesta: No (0)
¿Vas a insertar o eliminar elementos con frecuencia en medio de la colección?
Respuesta: No (0)
¿Los elementos se deben procesar en el mismo orden en que llegan (FIFO)?
Respuesta: Sí (1)
¿Los elementos se deben procesar en orden inverso a su llegada (LIFO)?
Respuesta: No (0)
¿Necesitas que los datos se mantengan ordenados automáticamente?
Respuesta: No (0)
¿Siempre necesitas extraer primero el elemento con mayor prioridad?
Respuesta: No (0)
¿Tu problema modela entidades con relaciones complejas (grafo)?
Respuesta: No (0)
¿Esperas manejar una cantidad grande y cambiante de datos?
Respuesta: Sí (1) (pueden llegar muchas llamadas)
Estas entradas alimentan la lógica de decisión implementada en main.c.

3. Decisión del asistente
Con esas respuestas, la parte relevante de la lógica es:
La pregunta 3 indica que las llamadas se procesan en orden FIFO.
No se requiere prioridad especial ni LIFO.
No hay relaciones tipo grafo ni necesidad de ordenamiento automático.
En el código en C, la condición clave es:
/* 3. FIFO -> Cola */
if (a->need_fifo && !a->need_lifo) {
    return STRUCT_QUEUE;
}

Dado que:
need_fifo = 1
need_lifo = 0
El asistente recomienda la estructura:
Cola (Queue)

4. Estructura recomendada: Cola (Queue)
La cola representa la fila de llamadas:
front -> [ Llamada 1 ] -> [ Llamada 2 ] -> [ Llamada 3 ] -> ... -> [ Llamada n ] <- rear

front: llamada que será atendida primero.
rear: última llamada que se agregó a la fila.

5. Pseudocódigo aplicado al caso del call center
5.1 Operación enqueue (llega una nueva llamada)
Descripción:
Agregar una nueva llamada al final de la cola.

Pseudocódigo:
function enqueue(Q, llamada):
    Q[rear] = llamada
    rear = (rear + 1) mod capacidad

Precondición: la cola no está llena.
Postcondición: la nueva llamada está al final de la fila.
Complejidad: O(1)

5.2 Operación dequeue (un agente atiende una llamada)
Descripción:
Tomar la llamada que lleva más tiempo esperando.

function dequeue(Q):
    if front == rear:
        error "La cola está vacía"
    llamada = Q[front]
    front = (front + 1) mod capacidad
    return llamada


Precondición: la cola no está vacía.
Postcondición: se devuelve la llamada más antigua y se elimina de la fila.
Complejidad: O(1)

6. Ejecución paso a paso del ejemplo
Supongamos:
Capacidad de la cola: 5.
Llamadas que llegan, en orden:
C1 (cliente 1)

C2 (cliente 2)

C3 (cliente 3)

Paso 1: llegan tres llamadas
enqueue(Q, C1)

enqueue(Q, C2)

enqueue(Q, C3)

Estado de la cola:
front -> [ C1 ] -> [ C2 ] -> [ C3 ] <- rear

Paso 2: un agente queda libre y atiende una llamada
llamada_atendida = dequeue(Q)

El sistema devuelve C1 (la primera que llegó).

Estado de la cola:

front -> [ C2 ] -> [ C3 ] <- rear

Paso 3: llegan dos llamadas más

enqueue(Q, C4)

enqueue(Q, C5)

Estado de la cola:

front -> [ C2 ] -> [ C3 ] -> [ C4 ] -> [ C5 ] <- rear

El orden de atención será: C2, C3, C4, C5.

7. Justificación de por qué una cola es adecuada
En este problema del call center:
Cada llamada debe ser atendida en el orden en que llega.
No se requiere atender primero “la más importante” (no hay prioridad especial).
No se necesita procesar la última llamada primero (no es LIFO).
No importa la posición exacta por índice, solo la posición relativa en la fila.
Por eso, una cola (queue):
Modela de forma natural la fila de clientes.
Mantiene el comportamiento FIFO.
Permite insertar y extraer llamadas en O(1).
Es fácil de implementar con un arreglo circular o una lista ligada.y
El asistente de estructuras de datos identifica estas características a través del cuestionario y recomienda la cola como estructura de datos principal para el sistema de atención de llamadas.
